## Проект в рамках изучения AQA (jsonplaceholder.typicode.com)

Учебный pet-project для отработки **архитектуры API-автотестов** и базовых инженерных практик AQA.  
Проект сознательно делается простым по стеку, но корректным по структуре и разделению ответственности.

В качестве тестового стенда используется публичный API:  
https://jsonplaceholder.typicode.com

---

## Стек

- Python 3.x  
- pytest  
- requests  

---

## Структура проекта

```text
learningAQA-pet-project/
├── client_api/          # HTTP-клиент (работа с requests)
│   └── client_api.py    # ClientAPI: base_url, Session, отправка запросов
│
├── assertions/          # Проверки и вспомогательные ассерты
│   └── assertions.py
│
├── payloads/            # Тела запросов / тестовые данные
│   └── posts_payloads.py
│
├── tests/               # Автотесты
│   ├── conftest.py      # pytest-фикстуры (client, base_url и т.д.)
│   └── test_posts.py    # Тесты доменного API
│
├── .gitignore
├── README.md
```

---

## Архитектурная идея

Проект построен по принципу **разделения ответственности:**

 - ClientAPI
    - в клиенте описано как делать HTTP запросы
    - инкапсуляция requests.Session
 - Доменные API (PostsAPI и т.д.)
    - в клиентах описано какой именно эндпоинт вызывать
    - используют ClientAPI, но не зависят от requests
 - Тесты
    - общаются только с доменными API
    - объект создается в фикстуре, которая потом используется
    - так же используют payload, которые сохранены в виде переменных и функций в отдельном файле

### Запуск тестов

1) Установить зависимости:

``pip install -r requirements.txt``

2) Запустить тесты:

``pytest -v``
